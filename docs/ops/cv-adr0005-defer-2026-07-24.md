## 3. MVP 成功标准（Accepted 后才追；本波不验收实现）

摘自 ADR 0005 D6 + `docs/plans/u3-mvp-checklist.md`：

- 用户能：浏览主题（产品线 arina-only + 用户 catalog）、一键 apply、见 kick 结果、见 **fresh 布尔**
- 全程 **仅一个** watch injector（`doctor` 可证 · `injectorPathFreshness.fresh`）
- 卸载壳 ≠ 必须卸皮；显式「恢复原生」才走 restore
- 平台：**Windows 第一**；macOS 壳不在 0005 范围
- 安全：loopback + 短命 token；主题导入同核 schema/对比度/路径校验

**本波对上表的态度：** 只冻结标准，**不**跑壳验收、**不**新建 prove 脚手架。现有 `shell-probe-client.mjs` 属尖兵探测面，**不是** U3 交付。

## 5. 何时可 reopen（门闩 · 全满足）

1. **人批：** 用户/总控显式 **Accept ADR 0005**（改 ADR 状态 + overview 挂链），不是 agent 自批  
2. **前置仍绿：** 0004 契约面（`npm run test:contracts` 等）保持可用；新 API 先 contracts  
3. **书面切片：** MVP 清单冻结（list/apply/doctor.fresh/kick · 单 watch）；排期/卡号可追溯  
4. **形态不漂：** 仍服从 PROJECT §1.5 — 壳只做 L1 管理器，**不**替换侧车/watch 核、不改 asar  
5. **开工形态：** 优先最小客户端（可先 Node 尖兵）→ 再可选 Tauri 2；**禁止**借壳引入第二 CDP 守护  

任一未满足 → **继续 DEFER**。  
scout 派发原文：U1 契约已绿但仍 **不 Accepted 不写壳代码**；本波停在文档。
